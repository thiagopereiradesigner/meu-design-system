⚡ Como Funciona o Virtual Scrolling

  • Renderiza apenas o visível: Apenas ~15-20 linhas são criadas no DOM
  • Scroll ilimitado: Pode ter 10.000, 100.000 linhas sem lag
  • Performance constante: Mesma velocidade com 100 ou 1 milhão de linhas
  • Header fixo: Cabeçalho sempre visível ao rolar
  • Buffer inteligente: 5 linhas extras acima/abaixo para transição suave

Anatomia do Componente (HTML)

Tabela com virtual scrolling + status via Badge (pill).

  • div.table-container: contêiner onde o JS renderiza o conteúdo visível.
  • table (gerado em runtime pelo demo): thead + tbody.
  • Status: span.badge badge-success|badge-warning|badge-neutral dentro de células td.
  • Interação (demo): cabeçalho clicável para sort (opcional no projeto).

Como Usar (HTML)

No demo, o HTML da tabela é gerado dinamicamente. Você precisa do contêiner + instanciar o VirtualScrollTable.

{/* HTML + JS (min) */}
<div id="table-10000" class="table-container compact"></div>

<script>
  const data = generateData(10000);
  const table = new VirtualScrollTable('table-10000', data, { compact: true });
</script>

<!-- quando renderizar linhas, status deve usar: -->
<span class="badge badge-success">Ativo</span>
<span class="badge badge-warning">Férias</span>
<span class="badge badge-neutral">Inativo</span>

Como Usar (React)

Para células de status, use Badge com formato pill (radius full). No app, a renderização do virtual scroll deve preservar o mesmo componente para consistência visual. 

{/* JSX */}
import { Badge } from '@/components/ui/Badge';
