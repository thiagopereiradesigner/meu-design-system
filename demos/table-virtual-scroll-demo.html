Tabela — scroll virtual

Lista tabular com virtual scrolling: só as linhas visíveis entram no DOM, mantendo desempenho estável com milhares de registros. Cabeçalho fixo, ordenação opcional e status com contraste adequado (--ds-success-fg / --ds-warning-solid-fg).

Use em tabelas densas (filas, logs, diretórios). Em telas estreitas, o viewport usa clamp e cabeçalhos com área mínima de toque; role sem overflow horizontal indesejado.

Acessibilidade

  • Cabeçalhos: th com min-height: 44px e focus-visible para ordenação por teclado.
  • Contraste: texto de célula em --ds-fg; badges usam pares --ds-*-bg / --ds-*-fg (ou --ds-warning-solid-fg sobre fundo de aviso).
  • Leitura + zoom: fontes em tokens --ds-font-size-*; contêiner com altura fluida (sem valor fixo único em px).

Anatomia do Componente (HTML)

Estrutura gerada pelo demo de referência.

  • div.table-container (opcional compact): viewport com borda e altura fluida.
  • div.table-scroll-area: área com overflow: auto e scroll suave em touch.
  • div.table-virtual-body + table do cabeçalho + div das linhas visíveis.
  • Células de status: span.badge com badge-success / badge-warning / badge-neutral.

Como Usar (HTML)

Contêiner no markup; o script instancia VirtualScrollTable e preenche o DOM. Ajuste --vs-table-row-* no CSS se mudar a densidade das linhas.

<div id="table-10000" class="table-container compact"></div>

<script>
  const data = generateData(10000);
  const table = new VirtualScrollTable('table-10000', data, { compact: true });
</script>

<!-- Status nas células -->
<span class="badge badge-success">Ativo</span>
<span class="badge badge-warning">Férias</span>
<span class="badge badge-neutral">Inativo</span>

Como Usar (React)

Em produção, prefira uma lib de virtualização (TanStack, etc.) e o mesmo sistema de tokens para linhas e badges.

import { Badge } from '@/components/ui/Badge';

Como funciona o virtual scrolling

  • Só o visível: cerca de 15–25 nós de linha no DOM, conforme altura do viewport.
  • Escala: dezenas de milhares de linhas sem custo proporcional de layout.
  • Cabeçalho: position: sticky dentro da área de rolagem.
  • Buffer: linhas extras acima/abaixo para transição suave ao rolar.
