TP.IA - Design System

Toast Component com WCAG AA

Notificações (toasts) com feedback visual e comportamento acessível (role/ARIA + teclado).

♿ Melhorias de Acessibilidade Aplicadas

  • Container do toast com role="alert", aria-live e aria-atomic.
  • Erro usa aria-live="assertive" para priorizar leitura.
  • Fechamento via botão com aria-label e via teclado com Esc.
  • Auto-close e pausa ao passar o mouse (progress bar).
Anatomia do Componente (HTML)
  • div#toast-container: região viva do toast (role="alert" + aria-live).
  • div.toast: wrapper do toast injetado via JS.
  • div.toast-content: agrupa p.toast-title e p.toast-message.
  • button.toast-close: botão para fechar (tem aria-label).
  • div.toast-progress + div.toast-progress-bar: visual de tempo restante.
  • Classes toast-success-active, toast-error-active, toast-warning-active, toast-info-active controlam cores/FG.
Como Usar (HTML)

Estruture a região do toast e chame showToast(type, variant) 

<div id="toast-container" class="toast-container"
     role="alert"
     aria-live="polite"
     aria-atomic="true"></div>

<button onclick="showToast('success', 'active')">Sucesso</button>

<script>
  function showToast(type, variant) {
    const container = document.getElementById('toast-container');
    const config = {
      success: { title: 'Sucesso!', message: 'Operação concluída.', duration: 3000 },
      error: { title: 'Erro!', message: 'Tente novamente.', duration: 5000 },
      warning: { title: 'Atenção!', message: 'Revise os dados.', duration: 4000 },
      info: { title: 'Info', message: 'Atualização disponível.', duration: 4000 }
    };

    const c = config[type] || config.info;
    container.innerHTML = `
      <div class="toast toast-${type}-${variant}">
        <div class="toast-content">
          <p class="toast-title">${c.title}</p>
          <p class="toast-message">${c.message}</p>
        </div>
        <button class="toast-close" onclick="hideToast()" aria-label="Fechar notificação">Fechar</button>
        <div class="toast-progress">
          <div class="toast-progress-bar" style="animation-duration:${c.duration}ms"></div>
        </div>
      </div>
    `;
    container.setAttribute('aria-live', type === 'error' ? 'assertive' : 'polite');
    container.classList.add('show');
    setTimeout(hideToast, c.duration);
  }

  function hideToast() {
    const container = document.getElementById('toast-container');
    container.classList.remove('show');
    setTimeout(() => (container.innerHTML = ''), 300);
  }
</script>
Como Usar (React)

Exemplo com ToastProvider + hook useToast. 

import React from 'react';
import { ToastProvider, useToast } from '@/components/Toast/ToastComponent';

function ToastButtons() {
  const { showToast } = useToast();

  return (
    <button
      onClick={() =>
        showToast({
          type: 'success',
          variant: 'active',
          title: 'Operação concluída!',
          message: 'Seus dados foram salvos com sucesso.',
        })
      }
    >
      Mostrar toast
    </button>
  );
}

export default function ToastPage() {
  return (
    <ToastProvider>
      <ToastButtons />
    </ToastProvider>
  );
}
type
'success' | 'error' | 'warning' | 'info' (define cores e aria-live).
variant
'active' ou 'light' (define palette do toast).
title
Título do toast (renderizado em p.toast-title).
message
Mensagem do toast (renderizada em p.toast-message).
duration
Duração (ms) para auto-close (opcional).

🎨 Active (Fundo Colorido)

Texto branco em fundo colorido - Todos passam WCAG AA (4.5:1+)

💡 Light (Fundo Claro)

Texto colorido em fundo claro - Menos chamativo

✨ Features Implementadas

  • Animação suave: Fade + Slide from top (40px)
  • Auto-close: 3s (success), 4s (info/warning), 5s (error)
  • Progress bar: Animada, pausa no hover
  • Hover pausa timer: Usuário pode ler com calma
  • Esc fecha: Acessibilidade de teclado
  • 1 toast por vez: Novo substitui o anterior
  • WCAG AA: Contraste mínimo 4.5:1
  • ARIA: role="alert", aria-live
