TP.IA - Design System

Componentes Input e Select

Demonstração de Input (texto) e Select (dropdown) com estados (normal, erro e disabled) e busca no dropdown.

♿ Melhorias de Acessibilidade Aplicadas

  • Use field-label com field-label-text para fornecer contexto textual ao campo.
  • Para erro, aplique .error no container correspondente (ex.: .field-input-container e helper).
  • Para desabilitado, use disabled (e mantenha o visual coerente em .disabled).
  • Dropdown deve fechar ao selecionar e não pode abrir quando está desabilitado (lógica via JS).
Anatomia do Componente (HTML)
  • div.field: wrapper do campo (Input ou Select).
  • div.field-label + span.field-label-text: rótulo (com opcional span.field-sublabel).
  • .field-input-container: container visual e também “trigger” do Select via onclick.
  • input.field-input: campo nativo do Input (foco/teclado funcionam naturalmente).
  • span.select-text: texto do valor selecionado (atualizado por selectOption).
  • div.select-dropdown: painel das opções (visibilidade via .open e id).
  • div.select-option (+ .disabled): cada opção do dropdown.
  • JS: toggleDropdown, selectOption e filterOptions (para o modo com busca).
Como Usar (HTML)

Estruture o trigger e o dropdown do Select usando os mesmos ids do JS do demo. 

<div class="field">
  <div class="field-label">
    <span class="field-label-text">Estado</span>
    <span class="field-sublabel">obrigatório</span>
  </div>

  <div class="select-container">
    <div class="field-input-container" onclick="toggleDropdown('estado')">
      <span class="select-text" id="estado-text">Selecione um estado</span>
      <span>▾</span>
    </div>

    <div class="select-dropdown" id="estado-dropdown">
      <div class="select-option" onclick="selectOption('estado', 'São Paulo', this)">
        <span>São Paulo</span>
      </div>
      <div class="select-option" onclick="selectOption('estado', 'Rio de Janeiro', this)">
        <span>Rio de Janeiro</span>
      </div>
    </div>
  </div>
</div>

<script>
  function toggleDropdown(selectId) { /* abra/feche dropdown + atualize classes */ }
  function selectOption(selectId, value, element) { /* atualize select-text e feche */ }
</script>
Como Usar (React)

Exemplo React controlado para um Select com as mesmas classes do demo. 

import React, { useMemo, useState } from 'react';

export function SelectField({
  label,
  sublabel,
  options,
  placeholder = 'Selecione...',
  helperText,
  error = false,
  disabled = false,
}) {
  const [open, setOpen] = useState(false);
  const [value, setValue] = useState('');

  const selectedLabel = useMemo(() => {
    return options.find((o) => o.value === value)?.label || placeholder;
  }, [options, value, placeholder]);

  return (
    <div className="field">
      <div className="field-label">
        <span className="field-label-text">{label}</span>
        {sublabel ? <span className="field-sublabel">{sublabel}</span> : null}
      </div>

      <div className="select-container">
        <div
          className={"field-input-container" + (error ? " error" : "") + (disabled ? " disabled" : "")}
          onClick={() => !disabled && setOpen((v) => !v)}
        >
          <span className={"select-text" + (value ? " has-value" : "")}>{selectedLabel}</span>
          <span>▾</span>
        </div>

        <div className={"select-dropdown" + (open ? " open" : "")}>
          {options.map((opt) => (
            <div
              key={opt.value}
              className={"select-option" + (opt.disabled ? " disabled" : "")}
              onClick={() => {
                if (disabled || opt.disabled) return;
                setValue(opt.value);
                setOpen(false);
              }}
            >
              <span>{opt.label}</span>
            </div>
          ))}
        </div>
      </div>

      <div className={"field-helper" + (error ? " error" : "")}>
        <span>{helperText || 'Selecione uma opção'}</span>
      </div>
    </div>
  );
}
options
Array com {`{ value, label, disabled? }`}.
placeholder
Texto quando nenhum item estiver selecionado.
error
Quando true, aplica classe error no trigger e helper.
disabled
Quando true, bloqueia abertura e aplica disabled visual.

♿ Acessibilidade

  • Focus ring visível em todos os estados (3px verde)
  • Estados de erro claramente identificados com cor e ícone
  • Helper text fornece contexto adicional
  • Placeholders descritivos
  • Contrast ratio conforme WCAG AA (4.5:1)
  • Ícones informativos para ajudar o usuário
